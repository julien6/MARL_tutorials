# Contributing to MARL Tutorials

Thank you for your interest in contributing to the MARL Tutorials project! This document provides guidelines for contributing.

## How to Contribute

There are many ways to contribute:

1. **Report Issues**: Found a bug or typo? Open an issue!
2. **Improve Documentation**: Clarify explanations, fix typos, add examples
3. **Add Exercises**: Create new practice problems
4. **Enhance Code**: Improve implementations, add features
5. **Share Extensions**: Add new notebooks or topics

## Getting Started

1. **Fork the Repository**
   ```bash
   git clone https://github.com/julien6/MARL_tutorials.git
   cd MARL_tutorials
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make Changes**
   - Follow existing code style
   - Test your changes
   - Update documentation if needed

4. **Commit Changes**
   ```bash
   git add .
   git commit -m "Description of changes"
   ```

5. **Push and Create Pull Request**
   ```bash
   git push origin feature/your-feature-name
   ```

## Contribution Guidelines

### Code Style

- **Python**: Follow PEP 8 style guide
- **Docstrings**: Use Google style docstrings
- **Type Hints**: Use type hints where appropriate
- **Comments**: Explain complex logic, not obvious code

### Notebook Guidelines

When creating or modifying notebooks:

1. **Clear Structure**
   - Title and overview at the top
   - Learning objectives
   - Prerequisites
   - Summary at the end

2. **Markdown Cells**
   - Use headers to organize sections
   - Provide clear explanations before code
   - Include mathematical notation where helpful

3. **Code Cells**
   - Keep cells focused and readable
   - Add comments for complex operations
   - Print intermediate results
   - Use meaningful variable names

4. **Visualization**
   - Include plots and visualizations
   - Label axes and add titles
   - Use consistent color schemes

5. **Exercises**
   - Provide exercises of varying difficulty

### Testing

Before submitting:

- [ ] Run all cells in notebooks
- [ ] Check for errors or warnings
- [ ] Verify visualizations display correctly
- [ ] Test on a fresh environment if possible
- [ ] Check that links work

### Documentation

- Update README.md if adding new features
- Document new functions and classes
- Add references for algorithms or papers
- Update course outline if adding notebooks

## Types of Contributions

### Bug Reports

When reporting bugs, please include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details (OS, Python version, etc.)
- Screenshots if applicable

### Feature Requests

When requesting features:
- Describe the feature clearly
- Explain why it would be useful
- Provide examples if possible

## Notebook Contributions

### Adding a New Notebook

If you want to add a new notebook:

1. **Proposal**: Open an issue describing the notebook
   - Topic and learning objectives
   - How it fits in the course sequence
   - Prerequisites and dependencies

2. **Structure**: Follow the template structure
   - See existing notebooks for format
   - Include theory, implementation, exercises

3. **Code Quality**: Ensure high quality
   - Test all code cells
   - Provide clear explanations
   - Include visualizations

4. **Integration**: Update documentation
   - Add to course outline
   - Update notebook index
   - Add to README if appropriate

### Improving Existing Notebooks

- Fix errors or typos
- Improve explanations
- Add visualizations
- Enhance exercises
- Update deprecated code
- Add references

## Review Process

1. **Submission**: Create a pull request
2. **Review**: Maintainers will review your contribution
3. **Feedback**: Address any comments or requests
4. **Merge**: Once approved, your contribution will be merged

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers
- Accept constructive criticism
- Focus on what's best for the community
- Show empathy

### Unacceptable Behavior

- Harassment or discriminatory language
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information
- Unprofessional conduct
